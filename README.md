# hubdl

Терминальная альтернатива Cloud Downloader: приём материалов с Яндекс Диска (публичные ссылки и WebDAV), Dropbox и Google Drive — без веб-интерфейса, launcher'а и tray. Одна команда, меню на `fzf`.

## Возможности

- скачивание по публичной ссылке `disk.yandex.ru` / `yadi.sk` — файлы и папки (папки качаются пофайлово, без лимита на zip)
- скачивание по пути или полному URL внутри WebDAV (через `rclone`, с параллельными загрузками и прогрессом)
- публичные ссылки Dropbox (папка приходит архивом — предложит распаковать)
- публичные файлы Google Drive
- прямые `http(s)`-ссылки
- прогресс по каждому файлу: проценты, объём, скорость и ETA (для WebDAV — стандартный прогресс rclone)
- докачка после обрыва: при потере соединения несколько автоматических повторов с того же места; повторный запуск той же ссылки пропускает готовые файлы и докачивает недокачанные
- проекты: каждая загрузка складывается в папку проекта внутри общей папки загрузок
- настройки в `~/.config/hubdl/config` (права `600`)

## Установка

```bash
cd hubdl && ./setup.sh
```

Установщик проверит зависимости (`fzf`, `jq`, опционально `rclone` — нужен только для WebDAV), поставит недостающее через Homebrew и добавит команду `hubdl` в `~/.local/bin`.

## Использование

```bash
hubdl                       # интерактивное меню
hubdl <ссылка> [проект]     # быстрая загрузка без меню
hubdl settings              # настройки
```

При первом запуске спросит папку для проектов и (по желанию) доступы WebDAV.

В меню:

- **Новая загрузка** — выбери проект (или создай новый), вставь ссылку или путь WebDAV, источник определится автоматически
- **Настройки** — папка проектов, WebDAV URL/логин/пароль, число одновременных загрузок
- **Открыть папку проектов** — открывает Finder

## Telegram-мост

Пункт меню **«Telegram мост»** (или `hubdl bridge`) превращает hubdl в агента для bridge-сервера из Editor_hub на VPS — того же, с которым работает веб-версия.

Настройка (меню «Настройки» или при первом запуске моста):

- **Bridge URL** — адрес bridge-сервера (`TASK_BRIDGE_URL`)
- **Bridge secret** — общий секрет (`BRIDGE_SHARED_SECRET` с VPS)
- **Имя агента** — как этот компьютер будет виден в боте (по умолчанию hostname)

Пока мост запущен:

- ссылка, отправленная боту, скачивается на этот компьютер; бот показывает живой прогресс: проценты, объём, скорость и ETA
- `/storage` в боте показывает свободное место и папку загрузок
- `/delete` удаляет проект с диска через кнопки
- отмена активной загрузки из бота останавливает скачивание

Ctrl-C останавливает мост и возвращает в меню. Локальные загрузки через меню и мост используют одни и те же настройки и папку проектов.

## Важно

- Для WebDAV Яндекс Диска нужен **пароль приложения**, не пароль аккаунта.
- Публичная ссылка может быть недоступна, если владелец запретил скачивание.
- Папки Google Drive без авторизации не скачиваются — настрой remote в `rclone config` или качай файлы по одному.

## Удаление

```bash
./uninstall.sh
```

## Обновление

```bash
git pull
```

Symlink указывает на папку проекта, поэтому после `git pull` команда сразу использует новую версию.
