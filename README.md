# hubdl

Терминальный загрузчик материалов для монтажа: Яндекс Диск (публичные ссылки и WebDAV), Dropbox, Google Drive и прямые ссылки. Никаких веб-интерфейсов и фоновых сервисов по умолчанию — одна команда и меню на `fzf`. Терминальная альтернатива Cloud Downloader (Editor_hub) в духе [studio-tools](https://github.com/cutcodeenv-tech/studio-tools).

## Возможности

- **Яндекс Диск, публичные ссылки** (`disk.yandex.ru`, `yadi.sk`) — файлы и папки; папки качаются пофайлово рекурсивно, поэтому нет лимита Яндекса на zip больших папок
- **Яндекс Диск, WebDAV** — по пути или полному URL, через `rclone` с параллельными загрузками
- **Dropbox** — публичные ссылки; папка приходит архивом, hubdl предложит распаковать
- **Google Drive** — публичные файлы
- **Прямые `http(s)`-ссылки**
- **Прогресс по каждому файлу**: проценты, объём, скорость, ETA
- **Докачка после обрыва**: авто-повторы с того же места; повторный запуск той же ссылки пропускает готовые файлы и докачивает недокачанные
- **Проекты**: каждая загрузка складывается в свою папку внутри общей папки загрузок
- **Telegram-мост**: ссылку можно прислать боту, и она скачается на этот компьютер — с живым прогрессом в чате; мост может работать в фоне и подниматься при входе в систему

## Требования

- macOS
- [Homebrew](https://brew.sh) — для установки зависимостей
- `fzf`, `jq` — поставит установщик
- `rclone` — опционально, нужен только для WebDAV (установщик предложит)

## Установка

```bash
git clone https://github.com/cutcodeenv-tech/Downloader.git
cd Downloader
./setup.sh
```

Установщик проверит зависимости, поставит недостающее через Homebrew и добавит команду `hubdl` в `~/.local/bin`. После установки перезапусти терминал (или `source ~/.zshrc`).

## Быстрый старт

```bash
hubdl
```

При первом запуске hubdl спросит:

1. **Папку для проектов** — куда складывать загрузки (внутри неё каждая загрузка получает свою папку-проект)
2. **WebDAV URL / логин / пароль** — можно пропустить (Enter), если WebDAV не нужен
3. **Число одновременных загрузок** — для WebDAV

Дальше — меню: выбери «Новая загрузка», укажи проект (или создай новый), вставь ссылку. Источник определяется автоматически.

## Команды

| Команда | Что делает |
|---|---|
| `hubdl` | интерактивное меню |
| `hubdl <ссылка> [проект]` | быстрая загрузка без меню (если проект не указан — спросит) |
| `hubdl settings` | настройки |
| `hubdl bridge` | запустить Telegram-мост в этом окне (Ctrl-C — остановить) |
| `hubdl bridge enable` | включить фоновый мост с автозапуском при входе в систему |
| `hubdl bridge stop` | закрыть фоновый мост до перезагрузки (автозапуск остаётся) |
| `hubdl bridge disable` | выключить фоновый мост насовсем |
| `hubdl bridge status` | статус моста и автозапуска |
| `hubdl bridge log` | хвост лога фонового моста |
| `hubdl update` | обновить hubdl до последней версии с GitHub |

## Настройки

Меню «Настройки» или `hubdl settings`. Хранятся в `~/.config/hubdl/config` (права `600`).

| Настройка | Описание |
|---|---|
| Папка проектов | корень, внутри которого создаются папки-проекты |
| WebDAV URL | по умолчанию `https://webdav.yandex.ru` |
| WebDAV логин | логин Яндекса (почта) |
| WebDAV пароль | **пароль приложения**, не пароль аккаунта — создаётся на [id.yandex.ru/security/app-passwords](https://id.yandex.ru/security/app-passwords) |
| Одновременных загрузок | параллельность для WebDAV (rclone) |
| Bridge URL | адрес bridge-сервера на VPS (для Telegram-моста) |
| Bridge secret | общий секрет с VPS (`BRIDGE_SHARED_SECRET`) |
| Имя агента | как этот компьютер виден в Telegram-боте (по умолчанию hostname) |

## Telegram-мост

Мост превращает hubdl в агента для bridge-сервера из Editor_hub на VPS — того же, с которым работает веб-версия. Пока мост запущен:

- ссылка, отправленная боту, скачивается на этот компьютер; бот показывает живой прогресс: проценты, объём, скорость и ETA
- `/storage` в боте показывает свободное место и папку загрузок
- `/delete` удаляет проект с диска через кнопки
- отмена активной загрузки из бота останавливает скачивание

### Настройка

1. На VPS должен работать `telegram-bridge-server` из Editor_hub (`npm run bridge:start`, в `.env`: `TELEGRAM_BOT_TOKEN`, `BRIDGE_SHARED_SECRET`, `BRIDGE_PORT`).
2. В hubdl (меню «Настройки» или при первом запуске моста) укажи:
   - **Bridge URL** — `http://адрес-vps:порт`
   - **Bridge secret** — тот же `BRIDGE_SHARED_SECRET`
   - **Имя агента** — любое, по нему выбираешь компьютер в боте
3. Запусти мост: пункт меню «Telegram мост» → «Запустить в этом окне», либо включи фоновый режим.

Несколько компьютеров могут быть агентами одновременно — бот спросит, какому отдать задачу.

### Фоновый режим (автозапуск)

Чтобы мост был на связи всегда — без открытого терминала и сразу после перезагрузки:

- **«Автозапуск в фоне: включить»** / `hubdl bridge enable` — мост регистрируется в launchd: работает в фоне, стартует при входе в систему, перезапускается при падении
- **«Закрыть мост (до перезагрузки)»** / `hubdl bridge stop` — закрыть сейчас; после перезагрузки поднимется сам
- **«Автозапуск в фоне: выключить»** / `hubdl bridge disable` — остановить насовсем

Технические детали:

- macOS не даёт launchd исполнять скрипты с внешнего диска, поэтому при включении код hubdl копируется в `~/.local/share/hubdl` и фоновый мост работает оттуда. После `git pull` выполни `hubdl bridge enable` ещё раз, чтобы обновить фоновую копию
- Настройки общие (`~/.config/hubdl/config`) — интерактивный и фоновый режим ведут себя одинаково; после смены настроек перезапусти фоновый мост (`enable` заново)
- Лог фонового моста: `~/Library/Logs/hubdl-bridge.log`
- Если папка загрузок на внешнем диске, при первой фоновой загрузке macOS может спросить разрешение на доступ к съёмным томам — разреши один раз
- Двойной запуск исключён: интерактивный мост откажется стартовать, пока работает фоновый, и наоборот

## Важно

- Для WebDAV Яндекс Диска нужен **пароль приложения**, не пароль аккаунта.
- Публичная ссылка может быть недоступна, если владелец запретил скачивание.
- Папки Google Drive без авторизации не скачиваются — настрой remote в `rclone config` или качай файлы по одному.

## Обновление

hubdl сам проверяет GitHub (фоновый `git fetch`, не чаще раза в 15 минут). Когда выходит новая версия, в главном меню появляется пункт **«⬆ Обновить hubdl — доступна новая версия»** — один Enter, и поставится последняя версия: `git pull`, обновление фоновой копии моста (если включён автозапуск) и перезапуск hubdl.

То же самое вручную: `hubdl update` или просто `git pull` в папке проекта.

## Удаление

```bash
./uninstall.sh
```

Уберёт команду и (по желанию) настройки. Если был включён фоновый мост, сначала `hubdl bridge disable`.

## Структура

```
bin/hubdl        — точка входа: меню, выбор проекта, определение источника
lib/ui.sh        — вывод, вопросы, fzf-меню
lib/config.sh    — настройки (~/.config/hubdl/config)
lib/progress.sh  — загрузка с прогрессом, докачкой и авто-повторами
lib/yandex.sh    — публичные ссылки Яндекс Диска (Cloud API)
lib/webdav.sh    — WebDAV через rclone
lib/dropbox.sh   — публичные ссылки Dropbox
lib/gdrive.sh    — публичные файлы Google Drive
lib/bridge.sh    — Telegram-мост: агент bridge-сервера + фоновый режим (launchd)
```
